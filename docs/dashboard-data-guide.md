# 대시보드 데이터 안내

이 문서는 각 화면이 어떤 데이터를 보여주는지 설명합니다.
개발자가 아니어도, 화면의 숫자가 어디서 오는지 이해할 수 있도록 적었습니다.

기준 문서: `docs/project-principles.md`

## SCM 화면

### 재고 현황

무엇을 보여주나:
- 지금 팔 수 있는 재고가 얼마나 있는지
- 어느 창고에 얼마나 쌓여 있는지

어떤 데이터를 쓰나:
- `mart_inventory_onhand`

### 품절 위험

무엇을 보여주나:
- 현재 판매 속도 기준으로 재고가 얼마나 버틸지
- 기준 일수보다 빨리 떨어질 가능성이 있는지

어떤 데이터를 쓰나:
- `mart_stockout_risk`

### 미입고 발주

무엇을 보여주나:
- 이미 발주했지만 아직 다 들어오지 않은 수량
- 발주 상태를 한눈에 보기 위한 요약

어떤 데이터를 쓰나:
- `mart_open_po`

## P&L 화면

### 매출

무엇을 보여주나:
- 기간별 매출
- 실제 수치인지 추정치인지

어떤 데이터를 쓰나:
- `mart_pnl_revenue`

### 매출원가

무엇을 보여주나:
- 출고 수량과 반품을 반영한 순수 원가

어떤 데이터를 쓰나:
- `mart_pnl_cogs`

### 영업이익

무엇을 보여주나:
- 매출에서 변동비와 고정비를 반영한 최종 이익

어떤 데이터를 쓰나:
- `mart_pnl_operating_profit`

## 화면이 같은 달 데이터를 고르는 방식

- SCM 화면은 날짜를 보고 해당 월 범위를 읽습니다.
- P&L 화면은 `period` 값을 보고 해당 월을 읽습니다.

## 이 문서를 볼 때 기억할 점

- 대시보드는 내부 원천 데이터 전체를 보여주지 않습니다.
- 화면에는 정리된 `mart` 데이터만 올립니다.
- 내부용 `core`, `raw`, `ops` 데이터는 직접 보여주지 않는 것이 원칙입니다.
