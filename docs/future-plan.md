# 전체 로드맵

프로젝트의 단계별 목표와 현재 진행 상황을 정리합니다.

---

## 진행 현황 한눈에 보기

| 단계 | 목표 | 상태 |
|------|------|------|
| Phase 1 | 핵심 화면 + 데이터 파이프라인 | 완료 |
| Phase 1.5 | 실무 사용 안정화 | 완료 |
| Phase 2 | SCM 핵심 지표 추가 | 완료 |
| Phase 3 | 관리자 운영 도구 | 완료 |
| Phase 4 | 분석 도구 고도화 | 완료 |
| Phase 5 | 자동 감지 + 알림 | 진행 중 (세션 1/4 완료) |

---

## Phase 1 — 핵심 화면 만들기 (완료)

재고와 손익 데이터를 올리고, 대시보드로 보는 기본 구조를 만들었습니다.

- 재고 현황, 품절 위험, 미입고 발주 화면
- 매출, 매출원가, 영업이익 화면
- CSV/엑셀 업로드 → 자동 처리 파이프라인

## Phase 1.5 — 실무에서 쓸 수 있게 다듬기 (완료)

현업 담당자가 실제로 편하게 쓸 수 있도록 안정화했습니다.

- 엑셀 다운로드 (필터 조건 반영)
- 창고/품목/채널 드롭다운 필터
- 큰 데이터 페이지 나눠서 보기
- CI/CD 자동 배포 + 에러 추적
- 기본 테스트

## Phase 2 — 의사결정에 필요한 지표 추가 (완료)

SCM 담당자가 판단할 때 꼭 필요한 분석 데이터를 추가했습니다.

- 재고회전율 (재고가 얼마나 빨리 도는지)
- ABC 분류 (중요한 품목 우선순위)
- 공급사 성적표 (납기, 품질, 가격 등)
- 채널별 수익성 비교
- 파이프라인 실행 결과 슬랙 알림

## Phase 3 — 관리자가 직접 운영하기 (완료)

개발자 없이도 관리자가 데이터 상태를 확인하고 문제를 처리할 수 있게 했습니다.

- 업로드 이력 조회 화면
- 데이터 재처리 (특정 단계만 다시 실행)
- 데이터 품질(DQ) 현황판
- 누가 언제 무엇을 했는지 감사 로그

## Phase 4 — 숫자에서 인사이트로 (완료)

단순히 숫자를 보여주는 것을 넘어, 원인을 파악하고 비교 분석할 수 있게 했습니다.

- 클릭으로 상세 데이터 탐색 (드릴다운)
- 지난달 대비 비교 분석
- 원하는 축으로 자유롭게 보는 리포트
- 병목 → 원인 → 대응 방안 연결

## Phase 5 — 시스템이 알아서 알려주기 (진행 중)

대시보드를 안 봐도 문제가 생기면 시스템이 알아채고 알려줍니다.

- **세션 1: 이상치 자동 탐지 (완료)**
  - 출고량 급변, 반품 급증, 재고 이상, 매출 급락을 자동 감지
  - 대시보드 상단에 이상 신호 배너 표시
- 세션 2: 슬랙/이메일 알림 (예정)
- 세션 3: "이렇게 하세요" 추천 액션 (예정)
- 세션 4: 정기 데이터 자동 처리 (예정)

상세 가이드: `docs/phase5-session-guide.md`

## 완료 시 달성하는 것

- **Phase 2까지**: 실무에서 매일 쓰는 SCM 대시보드
- **Phase 3까지**: 개발자 없이 운영 가능
- **Phase 5까지**: 문제를 알아서 찾아주는 자동화 플랫폼
