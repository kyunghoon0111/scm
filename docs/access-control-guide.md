# 접근 권한 안내

이 문서는 누가 어떤 데이터를 볼 수 있어야 하는지 정리한 문서입니다.
기술적인 정책 이름보다, 실제로 어떤 사람에게 무엇이 보여야 하는지를 중심으로 설명합니다.

기준 문서: `docs/project-principles.md`

## 기본 원칙

- 누구나 보는 대시보드는 정리된 `mart` 데이터만 사용합니다.
- 내부 운영용 데이터는 아무나 볼 수 없게 막아야 합니다.
- 관리자 기능은 관리자만 사용할 수 있어야 합니다.

## 사용자 구분

### 익명 사용자

로그인하지 않은 사용자입니다.

보여도 되는 것:
- 대시보드에 필요한 공개 범위 데이터

보이면 안 되는 것:
- `core.fact_po`
- `ops.threshold_config`
- 관리자용 함수와 화면

### 일반 로그인 사용자

로그인했지만 관리자 권한이 없는 사용자입니다.

보여도 되는 것:
- 본인에게 허용된 대시보드 화면

보이면 안 되는 것:
- 관리자 전용 설정
- 사용자 목록 같은 관리 기능

### 관리자

시스템 운영과 점검을 담당하는 사용자입니다.

할 수 있어야 하는 것:
- 사용자 목록 확인
- 시스템 상태 점검
- 기간 마감 같은 운영 작업

## 역할을 읽는 순서

시스템은 아래 순서대로 역할을 판단합니다.

1. JWT 안의 `role`
2. `app_metadata.role`
3. `user_metadata.role`
4. 아무 값이 없으면 `readonly`

## 지금 꼭 확인할 점

운영 DB를 점검했을 때, 익명 사용자가 내부 운영 데이터 일부를 읽을 수 있었던 흔적이 있습니다.
그래서 아래 두 가지를 반드시 다시 확인해야 합니다.

- 운영 DB에 최신 권한 마이그레이션이 적용되었는가
- 익명 사용자가 내부 데이터에 더 이상 접근하지 못하는가
