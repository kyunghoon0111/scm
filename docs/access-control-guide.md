# 누가 무엇을 볼 수 있는지

사용자 유형별로 어떤 화면과 기능을 사용할 수 있는지 정리합니다.

---

## 사용자 유형

### 비로그인 사용자 (데모 모드)

로그인하지 않아도 대시보드를 볼 수 있습니다.

| 할 수 있는 것 | 할 수 없는 것 |
|--------------|-------------|
| 대시보드 화면 조회 | 데이터 업로드 |
| 이상 신호 배너 확인 | 관리자 설정 변경 |
| 엑셀 다운로드 | 파이프라인 실행/재처리 |

### 일반 로그인 사용자

로그인했지만 관리자 권한이 없는 사용자입니다.

| 할 수 있는 것 | 할 수 없는 것 |
|--------------|-------------|
| 대시보드 전체 조회 | 관리자 전용 설정 |
| 데이터 업로드 | 사용자 목록 확인 |
| 엑셀 다운로드 | 기간 마감/롤백 |

### 관리자

시스템 운영과 점검을 담당합니다.

| 할 수 있는 것 |
|--------------|
| 모든 대시보드 조회 |
| 데이터 업로드 + 재처리 |
| 파이프라인 실행/중단/롤백 |
| 사용자 목록 확인 |
| 기간 마감, 시스템 설정 |

---

## 역할 판단 순서

시스템은 아래 순서대로 사용자의 역할을 확인합니다.

1. 로그인 토큰(JWT) 안의 `role`
2. `app_metadata.role`
3. `user_metadata.role`
4. 위 세 가지 모두 없으면 → 읽기 전용(`readonly`)
