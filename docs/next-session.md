# 다음에 할 일

이 문서는 다음 작업자가 가장 먼저 읽는 안내문입니다.
지금 프로젝트가 어디까지 정리됐는지와, 다음으로 무엇을 해야 하는지만 간단히 적었습니다.

최종 갱신일: 2026-03-13

## 지금 상태

이 프로젝트는 재고와 손익 정보를 한눈에 보는 대시보드입니다.
지금 실제로 복구해 둔 화면은 아래 6개입니다.

- 재고 현황
- 품절 위험
- 미입고 발주
- 매출
- 매출원가
- 영업이익

업로드, 설정, 관리자 화면은 아직 본격 기능 화면이 아니라 안내 화면으로 남아 있습니다.

## 이미 정리된 것

- 화면은 Phase 1에 필요한 6개 카드만 남겨서 단순하게 정리했습니다.
- 데이터 조회 방식은 실제 DB 테이블 구조에 맞게 다시 맞췄습니다.
- 권한 판단 기준은 `JWT role`을 가장 먼저 보고, 없으면 사용자 메타데이터를 보도록 정리했습니다.
- 운영 DB에 적용되어야 할 마이그레이션과 점검 스크립트를 준비했습니다.
- 오래된 문서와 안 쓰는 프론트 파일 일부를 정리했습니다.

## 아직 남은 핵심 일

가장 중요한 남은 일은 운영 Supabase에 수정 내용을 실제로 반영하고, 권한이 의도대로 동작하는지 확인하는 것입니다.

특히 확인해야 할 점:

- 대시보드용 `mart` 데이터는 계속 보여야 합니다.
- 내부 운영용 데이터인 `core.fact_po`와 `ops.threshold_config`는 익명 사용자에게 보이면 안 됩니다.
- 관리자 전용 기능은 관리자만 실행할 수 있어야 합니다.

## 바로 실행할 일

운영용 비밀값이 있는 환경에서 아래 두 명령을 순서대로 실행합니다.

```powershell
python scripts/apply_migrations.py
python scripts/verify_supabase_access.py
```

정상 결과는 아래와 같습니다.

- 대시보드 데이터는 조회됨
- 내부 운영 데이터는 익명 사용자에게 차단됨
- 관리자 전용 함수는 비관리자에게 차단됨
