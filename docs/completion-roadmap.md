# Completion Roadmap

이 문서는 이 프로젝트가 "어디까지 가야 완성으로 볼 수 있는지"를 단계별로 정리한 로드맵이다.

핵심 기준:
- 단순히 화면이 보이는 상태를 완성으로 보지 않는다
- 실제 운영 데이터가 안정적으로 들어오고
- 권한과 배포가 안전하며
- 운영자가 문제를 스스로 추적하고 복구할 수 있어야 한다

## 완료 기준 한 줄 요약

이 프로젝트의 완성은 아래 4가지를 모두 만족할 때로 본다.

1. 데이터가 끊기지 않고 들어온다
2. 대시보드 숫자를 신뢰할 수 있다
3. 운영자가 직접 점검/복구할 수 있다
4. 분석이 실제 의사결정에 쓸 수준으로 충분히 깊다

## Phase 1

목표:
- 기본 운영 가능 상태 만들기

포함 범위:
- SCM / P&L 핵심 화면
- 로그인/권한 가드
- 업로드 -> raw -> core -> mart 파이프라인
- Supabase / Vercel / Railway 연결
- 기본 RLS / 접근 통제

완료 기준:
- 업로드부터 대시보드 반영까지 한 사이클 성공
- 익명 접근 차단 정상
- 배포 환경에서 로그인/조회/업로드 가능

현재 상태:
- 거의 완료
- 남은 건 배포 환경에서의 최종 안정화 확인과 반복 검증

## Phase 2

목표:
- 데이터 계약을 두껍게 만들어 분석 품질 높이기

포함 범위:
- 업로드 계약 확장
- sales / charge / shipment / return 연결 키 강화
- 재고/발주/입고 운영 컬럼 보강
- 템플릿, 매핑, 문서 동기화

완료 기준:
- 최소 컬럼뿐 아니라 운영 상세 컬럼도 받아서 적재 가능
- 매출/비용/반품/물류 연결 설명력이 좋아짐
- 상품/채널/국가/창고 기준 분석 정확도 향상

현재 상태:
- 설계와 1차 구현 반영 완료
- 남은 건 실제 운영 데이터로 검증하고 core/mart 계산식 보강하는 것

## Phase 3

목표:
- 운영 관리 기능 완성

왜 필요하냐:
- 지금은 파이프라인이 도는지 개발자나 로그를 봐야 한다
- 완성 단계에선 운영자가 화면에서 상태를 보고 대응할 수 있어야 한다

포함 범위:
- 업로드 이력/배치 이력 화면
- 실패 원인 표시
- 재처리 버튼
- 컬럼 매핑 관리 UI
- 잡 상태, step, 소요시간 표시
- 관리자용 점검 화면
- 감사 로그 / 누가 무엇을 올렸는지 추적

완료 기준:
- 운영자가 DB 직접 안 봐도 업로드 상태를 파악 가능
- 실패 시 재처리 경로가 화면에 있음
- 관리자 기능이 문서가 아니라 실제 UI로 제공됨

아직 없는 것:
- 공식 구현 시작 전

## Phase 4

목표:
- 분석 제품으로서 완성도 높이기

포함 범위:
- 비용 배부 로직 고도화
- 공헌이익 / 영업이익 설명력 강화
- SKU / 채널 / 국가 / 창고별 수익성
- 리드타임, 반품, 품절 원인 분석
- 상세 drill-down
- 다운로드 / 리포트 기능

완료 기준:
- 숫자만 보는 대시보드가 아니라 원인 분석이 가능
- 운영팀 / 사업팀 / 경영진이 같은 시스템을 목적별로 쓸 수 있음

## Phase 5

목표:
- 자동화와 의사결정 지원

포함 범위:
- 이상치 탐지
- 품절/반품/수익성 저하 알림
- 추천 액션
- 스케줄 배치 자동화
- 데이터 품질 경고
- 알림 연동

완료 기준:
- 문제가 생긴 뒤 보는 시스템이 아니라
- 문제를 먼저 알려주고 대응을 돕는 시스템이 됨

## 진짜 "완성"으로 보려면 남은 일

지금 기준으로 꼭 필요한 건 아래다.

### 1. 운영 안정화 마감

- Vercel / Railway / Supabase 배포값 정리
- 업로드와 후처리 반복 검증
- CORS / env / 권한 문제 제거
- 실패 시 timeout과 에러 표시 정리

### 2. 데이터 신뢰도 확보

- 실제 운영 파일로 업로드 검증
- mart 숫자 검증
- 대시보드 값과 원천값 대조
- 중복 적재/재처리 정책 정리

### 3. 운영 UI 완성

- 업로드 이력
- 실패 사유
- 재처리
- 매핑 관리
- 관리자 점검 화면

### 4. 분석 심화

- sales / charge / shipment / return 중심으로 Phase 2 컬럼 실사용
- 수익성/반품/리드타임 분석 고도화
- drill-down 화면 추가

### 5. 자동화

- 정기 실행
- 품질 검증
- 이상치 탐지
- 알림

## 추천 우선순위

완성까지 가는 가장 현실적인 순서는 이거다.

1. Phase 1 완전 마감
2. Phase 2 실제 운영 데이터 검증
3. Phase 3 운영 관리 화면 구축
4. Phase 4 분석 고도화
5. Phase 5 자동화 / 추천

## 결론

이 프로젝트는 아직 "완성"이 아니다.

지금은:
- Phase 1 거의 완료
- Phase 2 설계와 기초 반영 진행

완성까지 보려면 최소한:
- 운영 안정화
- 운영 관리 UI
- 분석 심화

까지는 가야 한다.
