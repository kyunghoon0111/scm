# 프로젝트 운영 기준

이 문서는 이 프로젝트를 어떤 원칙으로 유지할지 정한 가장 상위 기준입니다.
문서가 많아 보여도, 실제로는 아래 원칙 몇 가지만 기억하면 됩니다.

## 가장 중요한 원칙

- 대시보드는 필요한 정보만 간단히 보여준다.
- 내부 운영 데이터는 아무 화면에나 직접 연결하지 않는다.
- 권한은 보수적으로 잡고, 필요한 사람에게만 연다.
- 문서는 다음 작업자가 바로 이해할 수 있게 짧고 분명하게 쓴다.

## 지금 프로젝트 범위

현재 우선순위는 새 기능 추가가 아닙니다.
먼저 아래 6개 조회 화면을 안정적으로 유지하는 것이 목표입니다.

- 재고 현황
- 품절 위험
- 미입고 발주
- 매출
- 매출원가
- 영업이익

## 데이터 사용 원칙

- 화면은 가능하면 `mart`처럼 정리된 데이터를 본다.
- `raw`, `core`, `ops` 데이터는 내부 처리나 운영 점검용으로 둔다.
- 원천 데이터를 화면에서 바로 읽는 구조는 피한다.

## 권한 원칙

- 익명 사용자는 내부 운영 데이터를 보면 안 된다.
- 일반 사용자는 필요한 화면만 본다.
- 관리자만 운영 기능을 실행한다.
